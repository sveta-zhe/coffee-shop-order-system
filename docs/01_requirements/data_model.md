# Модель данных системы заказов

## Сущности (таблицы)

1. **`order`** — заказ
2. **`order_item`** — позиция в заказе (напиток + добавки)
3. **`product`** — продукт (напиток, добавка)
4. **`payment`** — платеж
5. **`user`** — пользователь (клиент или бариста)

## Связи между сущностями

- Один заказ → много позиций (`order 1--* order_item`)
- Один продукт → может быть во многих позициях (`product 1--* order_item`)
- Один заказ → один платеж (`order 1--1 payment`)
- Один пользователь → может создать много заказов (`user 1--* order`)
