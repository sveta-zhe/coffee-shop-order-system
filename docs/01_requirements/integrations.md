# Интеграции системы

## 1. С платежным шлюзом

**Тип:** Синхронный HTTP-запрос (REST API)  
**Протокол:** HTTPS  
**Формат данных:** JSON  

**Поток:**
1. Система отправляет запрос на оплату: `POST /v1/payments`
2. Платежный шлюз возвращает статус: `success`, `failed`, `pending`
3. Система обновляет статус заказа в зависимости от ответа

**Обработка ошибок:**
- При таймауте — повторить запрос 3 раза с интервалом 5 секунд
- При ошибке `failed` — вернуть клиенту сообщение с предложением оплатить на месте

## 2. С СМС-сервисом (уведомления)

**Тип:** Асинхронный (очередь сообщений)  
**Когда отправляется:** После изменения статуса заказа на "ready"

**Формат сообщения:**
```json
{
  "phone": "+7XXX",
  "text": "Ваш заказ готов! Заберите на стойке"
}
```
## 3. С кассовой системой (файловый обмен)

### Формат обмена
- **Протокол:** SFTP (SSH File Transfer Protocol)
- **Расписание:** Каждый час, в 5 минут после окончания часа (например, в 10:05, 11:05)
- **Направление:** Наша система → Кассовая система (односторонний обмен)

### Структура файлов

#### Файл 1: `sales_summary_YYYY-MM-DD_HH.csv`
Содержит сводку по продажам за последний час.

| Колонка | Тип | Описание | Пример |
| :--- | :--- | :--- | :--- |
| hour | datetime | Час, за который данные | 2026-04-04 10:00:00 |
| total_amount | decimal(10,2) | Сумма продаж за час | 12450.50 |
| orders_count | integer | Количество заказов | 23 |

**Пример содержимого:**
```csv
hour,total_amount,orders_count
2026-04-04 10:00:00,12450.50,23
2026-04-04 11:00:00,18320.00,31
