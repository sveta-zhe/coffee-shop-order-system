# План тестирования

## 1. Объект тестирования
Система управления заказами для сети кофеен «Кофеин».

## 2. Виды тестирования

| Вид | Что проверяем | Когда |
| :--- | :--- | :--- |
| **Модульное** | Отдельные функции (скидка, расчет суммы) | Во время разработки |
| **Интеграционное** | API + БД, API + платежный шлюз | После модульных тестов |
| **Нагрузочное** | Система под пиковой нагрузкой (0.056 RPS) | Перед релизом |
| **Приемочное** | Соответствие требованиям (user stories) | Перед сдачей заказчику |
| **Регрессионное** | Что старые фичи не сломались | После каждого изменения |

## 3. Сценарии для приемочного тестирования

### Позитивные сценарии (что должно работать)

| ID | Сценарий | Ожидаемый результат |
| :--- | :--- | :--- |
| **AT-01** | Клиент выбирает напиток, добавки, оплачивает картой | Заказ создан, статус "paid", уведомление отправлено |
| **AT-02** | Бариста видит очередь заказов на планшете | Новый заказ появляется в конце очереди |
| **AT-03** | Бариста отмечает заказ как "готово" | Статус заказа меняется, клиенту приходит уведомление |
| **AT-04** | Управляющий открывает дашборд | Видит выручку за сегодня, количество заказов |

### Негативные сценарии (обработка ошибок)

| ID | Сценарий | Ожидаемый результат |
| :--- | :--- | :--- |
| **AT-05** | Клиент заказывает напиток, которого нет в наличии | Система показывает "Временно нет в наличии" |
| **AT-06** | Оплата не прошла (ошибка платежного шлюза) | Клиент видит сообщение, заказ не создан |
| **AT-07** | Клиент не выбрал напиток и нажал "Оплатить" | Система подсвечивает ошибку, не создает заказ |
| **AT-08** | Бариста пытается отметить заказ как "готово", которого нет в очереди | Система возвращает ошибку 404 |

## 4. Инструменты для тестирования

| Инструмент | Для чего |
| :--- | :--- |
| **pytest** | Модульные и интеграционные тесты (Python) |
| **Postman / Apifox** | Тестирование API вручную |
| **JMeter** | Нагрузочное тестирование |
| **TestRail** | Управление тест-кейсами (опционально) |

## 5. Критерии приемки

Система считается готовой к релизу, если:
- Все позитивные сценарии (AT-01–AT-04) пройдены
- Все негативные сценарии (AT-05–AT-08) пройдены
- Нет критических багов (упала система, потеря данных)
- Нагрузочный тест показал: p95 latency < 500 мс при RPS = 0.056

## 6. Роли и ответственность

| Роль | Что делает |
| :--- | :--- |
| **Аналитик** | Пишешь сценарии, участвуешь в приемочном тестировании |
| **Разработчик** | Пишет модульные и интеграционные тесты |
| **Тестировщик** | Проводит ручное тестирование, заводит баги |
| **Заказчик** | Подписывает акт приемки после успешного тестирования |
