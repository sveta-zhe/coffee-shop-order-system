# Чек-лист для ручного тестирования

## Перед началом тестирования
- [ ] Установлена тестовая база данных
- [ ] Запущен API сервер
- [ ] Запущен платежный шлюз (mock)
- [ ] Подготовлены тестовые пользователи (клиент, бариста, управляющий)

## Тестирование API

### POST /orders
- [ ] Успешное создание заказа → 201 Created
- [ ] Не указаны обязательные поля → 400 Bad Request
- [ ] Неверный формат поля → 400 Bad Request
- [ ] Товара нет в наличии → 400 с сообщением

### GET /orders/{id}
- [ ] Существующий заказ → 200 OK с данными
- [ ] Несуществующий заказ → 404 Not Found
- [ ] Неверный формат ID → 400 Bad Request

### PATCH /orders/{id}/status
- [ ] Обновление статуса → 200 OK
- [ ] Неверный статус → 400 Bad Request
- [ ] Заказ не найден → 404 Not Found

## Тестирование очереди заказов
- [ ] При создании заказа карточка появляется у бариста
- [ ] При отметке "готово" карточка исчезает
- [ ] Заказы отображаются в порядке поступления (FIFO)
- [ ] При большом количестве заказов (10+) очередь не тормозит

## Тестирование дашборда
- [ ] Выручка обновляется после оплаты
- [ ] Счетчик заказов увеличивается
- [ ] Топ-3 напитков показывает правильные данные
- [ ] Дашборд доступен только управляющему
