# Пример баг-репорта для проекта «Кофеин»

## ID бага
BUG-001

## Заголовок
При заказе напитка с добавкой общая сумма не пересчитывается

## Серьезность
- [x] Высокий (High) — функция работает с ошибками, клиент видит неверную цену

## Приоритет
- [x] Высокий — исправить срочно

## Шаги для воспроизведения
1. Открыть страницу заказа
2. Выбрать напиток "Латте" (цена 180 руб.)
3. Добавить добавку "Сироп" (цена 50 руб.)
4. Посмотреть общую сумму в корзине

## Фактический результат
Общая сумма отображается 180 руб. (без учета добавки)

## Ожидаемый результат
Общая сумма должна быть 230 руб. (180 + 50)

## Окружение
- Браузер: Chrome 120
- ОС: Windows 11
- API: localhost:8000

## Приложения
[Скриншот: корзина с суммой 180 руб. при наличии добавки]

## Дополнительная информация
Проблема воспроизводится только при добавлении первой добавки.  
Если добавить вторую добавку — сумма пересчитывается правильно.
