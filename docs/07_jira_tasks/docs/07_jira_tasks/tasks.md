# Задачи для разработки (Jira)

## Epic: Система управления заказами

### Задача 1. Реализовать API для создания заказа (POST /orders)
- **Приоритет:** High
- **Оценка:** 8 story points
- **Описание:** см. [api_specification.yaml](../02_api/api_specification.yaml)
- **Критерии приемки:**
  - [ ] 201 при успехе
  - [ ] 400 при неверных данных
  - [ ] 400 при отсутствии продукта
  - [ ] 500 при ошибке платежного шлюза

### Задача 2. Реализовать API для получения заказа (GET /orders/{id})
- **Приоритет:** Medium
- **Оценка:** 3 story points
- **Описание:** см. [api_specification.yaml](../02_api/api_specification.yaml)
- **Критерии приемки:**
  - [ ] 200 с данными заказа
  - [ ] 404 если заказ не найден

### Задача 5. Реализовать очередь заказов для бариста
- **Приоритет:** High
- **Оценка:** 5 story points
- **Описание:** 
  - При создании заказа карточка добавляется в очередь
  - При нажатии "Готово" карточка исчезает
  - Очередь обновляется в реальном времени
- **Критерии приемки:**
  - [ ] Заказы отображаются в порядке FIFO
  - [ ] Карточка содержит номер, напитки, время
  - [ ] По нажатию "Готово" статус меняется
